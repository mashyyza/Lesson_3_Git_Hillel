List of commands when working in the terminal
Initial setting - Setting user information for all local repositories
cd [name folder] entering the folder
git config user.name see name on Git
git config --global user.name "[name]" change or created name
git config --global user.email "[email]" change or created email
git init intialized Git repository in local folder
git clone [url-адрес] Downloads the repository along with its entire history of changes
Making changes - Viewing changes and creating commits (fixing changes)
git status Lists all new or changed files that need to be fixed
git add [file name] Indexes the specified file for subsequent commit
git add . Edd all file wich was changed current folder
git add *css Edd all file wich was changed current folder with extension css
git commit -m "[message with description]" Fixes indexed changes and saves them in version history
git commit -a -m "[message with description]" Sequential execution git add and git commit. Only tracked file
git commit --amend -m "[new message with description]" Complements the last commit without creating a new one
git reset [file] Cancels the indexing of the specified file, while preserving its contents
git diff --stange shows the difference between the current tracked state of the repository and the last snapshot of the repository
Creating a repository - Creating a new repository or getting an ego from an existing URL
git init [project name] A new local repository with the given name is created
git clone [url-адрес] Downloads the repository along with its entire history of changes
View history - Viewing and studying the history of project file changes
git log History of commits for the current branch
git diff shows the difference between the current untracked state of the repository and the last snapshot of the repository
git diff COMMIT_ID shows the difference between the current repository and specified of the repository
Rollback commits - Error removal and correction of the created history
git reset --hard HEAD^^ Resets the entire history along with the state of the working directory to the specified commit
git reset --mixed HEAD^^ Returns the project to the specified commit and moving all commits to the untracked zone.
git reset --soft HEAD^^ Reverts the project to the specified commit, while setting all commits to tracked status.
Moving between commits
git checkout [has commit #2] or git checkout HEAD^^ or git checkout HEAD~2 Revert to commit 2
git checkout master Switch back to the current commit
Command to configure and view remote repositories
git remote -v Viewing the list of existing remote repositories
git remote add [repozitiri_name_repozitori_adress] Add a new remote repository located at the specified address
git remote remove [name_repozitori] Delet repositories with this name
Synchronization with a remote repository - Registration of a remote repository and exchange of changes
git push Uploads all changes of the local branch to the remote repository
git pull Downloads history from a remote repository and merges it with the local one. pull = fetch + merge
Collective work - Named series of commits and connection of work results
git branch A list of named commit branches with an indication of the selected branch
git branch [name branch] Created new branch
git branch -d [name branch] Delete the branch
git checkout [name branch] Switch to the specified branch
git checkout -b Go to the newly created branch
git merge [name branch] Changes the specified branch to the current branch
